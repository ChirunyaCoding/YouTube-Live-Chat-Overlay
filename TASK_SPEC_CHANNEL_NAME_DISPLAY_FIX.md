# タスク仕様: チャンネル名表示が反映されない不具合修正

## 背景
- popup の「名前の表示形式」で「チャンネル名（API使用）」を選択しても、表示名に切り替わらないケースがある。
- `content.js` 内にYouTube Data API取得処理は存在するが、メッセージ描画フローから呼ばれていない。

## スコープ
- `content.js`: メッセージ投入後に、必要条件を満たす場合のみAPIでチャンネル名を補完する。
- `message-flow.js`: 既に表示中の行・履歴メッセージの表示名を後から更新できるAPIを追加する。
- `manifest.json`: バグ修正に伴うPATCHバージョン更新。
- 回帰テストの追加と実行。

## 制約
- 既存の表示順・表示タイミングを壊さない（API待ちで描画遅延させない）。
- APIキー未設定時は既存挙動（ハンドル表示）を維持する。
- 不要なAPI呼び出しを抑える（表示名が既に確定している場合は補完をスキップする）。
- 拡張機能アイコン設定は追加しない。

## 受け入れ基準
- `authorNameMode = "display-name"` かつAPIキー有効時、表示名未解決メッセージはAPI結果で後追い更新される。
- API解決できない場合でも、表示は継続しクラッシュしない。
- 既存行の作者名が更新された場合、表示テキストへ反映される。
- 回帰テストが追加され、ローカル実行で成功する。
- Playwrightの実ブラウザ検証を1回以上実施し、致命的エラーがない。

## 非対象
- YouTube APIの高度な再試行制御・バックオフ戦略の導入。
- 名前表示モード以外のUI/スタイル仕様変更。
