# タスク仕様: 自分のコメントが2重表示される不具合修正

## 背景
- ライブチャットオーバーレイで、投稿直後の自分のコメントが同内容で2行表示されることがある。
- 現象として、仮レンダラー（アイコン未確定）と確定レンダラー（アイコン確定）が別ID扱いになっている可能性が高い。

## スコープ
- `parser.js` のメッセージID解決を強化し、DOM属性だけでなくRendererデータ由来の安定IDも優先利用する。
- 同一コメントが仮レンダラーと確定レンダラーで到着しても、同一IDとして重複投入を防ぐ。
- `tests/` に回帰テストを追加して再発防止する。
- `manifest.json` のバージョンをPATCH更新する。

## 制約
- 既存の表示順、フェードアウト、表示名補完フローには影響を与えない。
- APIキーや外部サービス設定は変更しない。
- 既存のID解決（`data-id` 等）が取得できるケースはそのまま優先する。

## 受け入れ基準
- 仮レンダラー（属性IDなし相当）で受信したコメントと、後続の確定レンダラー（属性IDあり）が同一IDとして扱われ、2重表示されない。
- 既存メッセージの通常表示（他ユーザーコメント含む）が維持される。
- 追加した回帰テストがローカル実行で成功する。

## 非対象
- チャット取得方式（MutationObserver/隠しiframe）の全面変更。
- 表示デザインやアニメーション仕様の変更。
- YouTube Data API 呼び出し仕様の変更。
